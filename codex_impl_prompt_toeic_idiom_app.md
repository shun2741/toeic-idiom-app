# Codex実装依頼プロンプト（TOEIC英熟語学習アプリ）

以下の仕様に基づいて、Webアプリを実装してください。

## 目的
TOEIC 700〜860点帯の学習者向けに、英熟語を自由入力で学習できるWebアプリを作成したいです。少人数に共有して使う前提で、本格的な商用サービスではありませんが、見た目や使い勝手は実在する一般向け教育サービスのように、自然で親しみやすく、安っぽく見えない品質にしてください。

## 重要方針
- MVPとして動く完成度を優先してください
- OpenAI APIへの依存は最小化してください
- 採点は毎回LLMに丸投げせず、まずルールベースで判定し、曖昧な場合だけサーバー側でOpenAI APIを呼んでください
- APIキーは必ずサーバー側にのみ保持し、クライアントから直接OpenAI APIを呼ばないでください
- 問題は毎回リアルタイム生成せず、まずはローカルJSONまたはDBの固定問題を中心に出題してください
- 少人数運用を想定し、シンプルで保守しやすい実装にしてください

## 技術スタック
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Supabase
  - Auth: Googleログイン
  - Database: Postgres
- OpenAI API（サーバー側のみ）
- Vercelでデプロイしやすい構成

## 必須機能
1. Googleログイン
2. TOEIC 700〜860点帯の英熟語問題の出題
3. 自由入力回答
4. 採点機能
5. 間違えた問題の復習機能
6. 学習履歴の保存
7. シンプルなダッシュボード

## 問題形式
まずは以下の1種類を実装してください。

### 日本語→英熟語入力
例:
- 問題: 「延期する」
- 正答: `put off`

余裕があれば将来追加しやすい設計にしてください。
将来的には英文穴埋めも追加したいですが、今は不要です。

## 採点仕様
採点は以下の3段階で実装してください。

### 1. 正規化
- 小文字化
- 前後空白除去
- 連続スペースの正規化
- 不要な記号の簡易吸収

### 2. ルールベース判定
- 完全一致
- 許容表記ゆれ一致
- 典型ミス（スペルミス1文字程度、前置詞違いなど）の簡易判定

### 3. OpenAIによる曖昧判定
以下の場合のみサーバー側でOpenAI APIを呼ぶ
- ルールベースで正誤が確定しない場合
- 意味は近いが表現が異なる可能性がある場合
- 部分点やフィードバック生成が必要な場合

### 採点結果
最低限以下を返してください。
- isCorrect: boolean
- score: number (0〜1)
- judgment: `correct` / `almost_correct` / `incorrect`
- correctAnswer: string
- feedbackJa: string
- errorTags: string[]

## キャッシュ方針
APIコスト削減のため、同じ問題ID + 同じユーザー回答の組み合わせについては、LLM採点結果をDBまたはサーバーキャッシュに保存し、再利用してください。

## 復習機能
以下の簡易復習ロジックを実装してください。
- incorrect → 1日後に再出題
- almost_correct → 2日後に再出題
- correct → 4日後に再出題
- 連続正解で間隔を伸ばす余地を残す

復習対象だけを出す画面も作ってください。

## データ構造
最低限、以下に対応する設計にしてください。
- idioms
- idiom_variants
- questions
- user_answers
- review_queue
- llm_judgment_cache

厳密なスキーマは実装しやすい形で調整して構いません。
SupabaseのSQLスキーマも用意してください。

## UI/UXデザイン要件
ここはかなり重視します。

以下の方向性でデザインしてください。

- 実在する一般向け教育サービスのような、親しみやすく信頼感のあるUI
- 派手すぎず、地味すぎず、万人受けするデザイン
- 余白を十分に取り、読みやすくする
- スマホでもPCでも自然に使えるレスポンシブ対応
- カードUIや柔らかい角丸を使って、学習アプリらしいやさしい印象にする
- フォントサイズ、行間、ボタンサイズは視認性重視
- 安っぽい管理画面風ではなく、学習継続したくなる雰囲気にする
- 配色はクリーンで教育サービスらしいものにする
  - ベースは白またはごく薄いグレー
  - アクセントは青、青緑、または落ち着いたインディゴ系
  - 正解・不正解表示は見やすいがきつすぎない色
- ローディング、採点結果表示、ボタン押下などに軽いアニメーションを入れてください
- ただし過剰な演出は不要です

デザインの印象としては、以下を目指してください。
- 大衆向け英語学習サービス
- ベネッセ、スタディサプリ、Duolingo、英会話アプリなどにありそうな自然な使いやすさ
- ただし特定サービスのコピーではなく、独自実装で問題ないです

## OpenAI採点の実装方針
- OpenAI SDKを利用
- サーバー側のAPIルートまたはserver actionから呼び出す
- 構造化されたJSONを返すように実装
- モデル名は環境変数で差し替え可能にする
- プロンプトは採点専用で簡潔に保つ
- 再現性のため、判定フォーマットは固定する
